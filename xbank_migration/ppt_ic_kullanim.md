# PPT İç Kullanım — Finhouse için X Bank Merchant Migration Değerlendirmesi

## Slayt 1 — Başlık
**X Bank Merchant Migration Fırsatı**  
Finhouse iç değerlendirme sunumu

## Slayt 2 — Konu Nedir?
- X Bank, Payten benzeri dış altyapıdan kendi sanal POS sistemine geçiyor
- İlk 60 merchant taşınmış durumda
- Sırada yaklaşık 18.000 merchant var
- Bu iş teknikten çok operasyon + migration programı

## Slayt 3 — Neden Fırsat?
- büyük ölçekli delivery işi
- bankacılık referansı yaratır
- uzun süreli operasyon geliri doğurabilir
- işlem bazlı gelir modeli varsa yüksek potansiyel
- migration sonrası ek ürün fırsatları doğabilir

## Slayt 4 — Neden Zor?
- merchant’ların çoğu teknik olarak homojen değil
- herkesin IT ekibi yok
- hazır paket, custom, ajans bağımlı yapılar var
- müşteriler geçişe zaman ayırmayabiliyor
- proje yönetimi kadar saha takibi gerekiyor

## Slayt 5 — Bizim Avantajımız
- payment / fintech bağlamını biliyoruz
- QNB benzeri migration deneyimi var
- operasyon + teknik + ticari dili birlikte kurabiliyoruz
- klasik outsource ekibinden farklı, model tasarlayabiliriz

## Slayt 6 — Araştırmadan Çıkan Ana Dersler
- wave bazlı geçiş şart
- büyük merchant’ta parallel run gerekli olabilir
- KOBİ tarafında plugin / hosted page / self-service çok kritik
- migration başarısı veri kalitesi + segmentasyon + destek modeline bağlı

## Slayt 7 — Merchant Segmentleri
- hazır paket kullananlar
- standart API kullananlar
- ajans/yazılım firması bağımlı olanlar
- teknik kapasitesi düşük KOBİ’ler
- düşük hacimli / pasif merchant’lar

## Slayt 8 — Kazanma Stratejimiz
- yalnızca insan kaynağı satmayalım
- migration factory yaklaşımı sunalım
- AI destekli delivery modeli önerelim
- hazır paketler için geçiş kitleri / plugin mantığı kuralım

## Slayt 9 — Minimum Kârlı Delivery Modeli
- çekirdek ekip + otomasyon
- migration portal
- readiness checker
- AI destekli support triage
- platform bazlı playbook ve kitler

## Slayt 10 — Maliyet Okuması
- önceki kaba modele göre maliyet yüksek görünüyordu
- yeni modelde baz senaryoda merchant başı maliyet ~61 USD
- 65 USD bandı mümkün ama marj sınırlı
- sadece setup fee ile gitmek riskli

## Slayt 11 — En Sağlıklı Ticari Model
- discovery / pilot sabit bedeli
- merchant başı dönüşüm bedeli
- işlem bazlı gelir (binde 2 gibi)
- gerekirse segment bazlı farklı ücretleme

## Slayt 12 — Kârlılığı Belirleyen Kritik Faktörler
- hazır paket merchant oranı
- veri kalitesi
- custom merchant oranı
- otomasyon seviyesi
- pilot öğrenimlerinin kalitesi
- proje süresi / aylık hedef yoğunluğu

## Slayt 13 — Minimum Veriyle Başlama Stratejisi
- X Bank’a ilk aşamada kısa veri talebiyle git
- profesyonel ve kontrollü dur
- güven oluşunca discovery fazında veri taleplerini büyüt

## Slayt 14 — İnsan Kaynağı Profili
- klasik çağrı merkezi değil
- klasik yazılım ekibi de değil
- hibrit profiller lazım:
  - operasyon + teknik okuryazarlık
  - payment domain farkındalığı
  - merchant iletişimi

## Slayt 15 — Ürünleşme Fırsatı
- Ticimax / IdeaSoft / WooCommerce / Shopify için geçiş paketleri
- bu araçlar başka bankalarda da kullanılabilir
- proje sadece hizmet değil, ürün tohumu da olabilir

## Slayt 16 — İç Karar Soruları
- bu işi hangi marjla almak isteriz?
- discovery ücretini şart koşacak mıyız?
- işlem gelirine girmeden projeye girer miyiz?
- hangi minimum veri gelmeden fiyat vermeyiz?
- geçiş portalını ne kadar hızlı ürünleştirebiliriz?

## Slayt 17 — Önerilen Sonraki Adım
- iç sunum üzerinden hizalanma
- X Bank’a gidecek sade deck’i ayırma
- minimum veri talebi setini netleştirme
- discovery fazını ticari olarak çerçeveleme
