## OpenClaw Derinlemesine İnceleme (devam)
Bugün okunan dokümanlar:
- **architecture.md** — Gateway tek WebSocket sunucusu, tüm kanallar (WhatsApp/Telegram/Discord/Signal/iMessage) buradan geçiyor. Nodes (iOS/Android/macOS) aynı WS'ye `role:node` ile bağlanıyor. Canvas da aynı portta çalışıyor (18789).
- **multi-agent.md** — Birden fazla agent aynı gateway'de izole çalışabilir: ayrı workspace, ayrı auth, ayrı session store. Bindings ile routing yapılıyor (peer/channel/account bazlı). Aynı WhatsApp numarasından farklı DM'ler farklı agentlara yönlendirilebilir.
- **features.md** — Desteklenen kanallar: WhatsApp, Telegram, Discord, iMessage, Mattermost (plugin), IRC, Signal, Slack, Line, Google Chat, MS Teams, Matrix, Nostr, vb. iOS/Android node desteği: kamera, ekran kaydı, konum, sesli komut.
- **agent.md** — pi-mono tabanlı embedded runtime. Workspace dosyaları (AGENTS.md, SOUL.md, USER.md, IDENTITY.md) ilk turn'de inject ediliyor. Sessionlar JSONL olarak saklanıyor. Queue modları: steer (tool call arası enjeksiyon), followup, collect.
- **session.md** — DM'ler varsayılan olarak tek main session'a collapse oluyor. `dmScope: per-channel-peer` ile kullanıcı bazlı izolasyon sağlanıyor. Gateway session state'in tek kaynağı.
- **skills.md** — 3 katmanlı yükleme: bundled → managed (~/.openclaw/skills) → workspace (en yüksek öncelik). ClawHub üzerinden skill keşfi ve kurulumu.
- **subagents.md** — Paralel iş için izole arka plan agent'ları. Kendi session'ları var, bitince requester chat'e sonuç duyuruluyor. Thread-bound persistent session desteği var.
